# “单细胞大模型文献简报”自动化追加说明

在完成每日检索与中文简报后，将结构化结果同步到 GitHub 仓库 `jianqing666/singlecell`，使 GitHub Pages `https://jianqing666.github.io/singlecell/` 自动更新。

## 仓库更新流程

1. 读取自动化记忆文件，确认上次检索窗口和已收录 DOI/链接。
2. 读取仓库 `data/briefs.json` 的最新内容。
3. 以 DOI 为第一去重键；无 DOI 时使用规范化原文链接，再无链接时使用标准化标题。
4. 新条目按既有 JSON 字段补全；不得删除历史条目。论文修订或正式发表时更新原条目，并在 `updateType` 中准确标注。
5. `categories` 可多选，但只能使用下列固定分类：
   - 单细胞基础模型
   - 虚拟细胞与扰动
   - 空间与时空组学
   - 多组学与调控
   - 细胞图谱与数据底座
   - 测序与方法学
6. `evidence` 只能是：`同行评审`、`预印本`、`观点与解读`。
7. `priority` 从 1 开始按当前重要性排序；历史条目可顺延，但不得因“今日无新增”而删除。
8. 更新 `updatedAt` 和 `statusMessage`，再写回 `data/briefs.json`。
9. 创建或更新 `reports/YYYY-MM-DD.md`，保存当日简报、检索来源、关键词和“无高相关更新”结论（如适用）。
10. 完成远程写入后，在 Codex 输出中明确列出：新增数、修订数、累计数、更新过的仓库文件和网页地址。

## 写作与事实要求

- 优先使用 DOI、PubMed、期刊页或预印本原始页面。
- 明确区分同行评审、预印本、评论/观点与代码数据发布。
- `summary` 只写核验过的事实；`relevance` 说明与单细胞大模型/虚拟细胞/测序方向的关联；`followUp` 给出具体后续观察点。
- 不夸大结论；无法从原始来源确认的细节不写入 JSON。
